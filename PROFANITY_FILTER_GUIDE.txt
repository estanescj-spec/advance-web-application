PROFANITY & BANNED WORDS FILTER
Pure JavaScript Implementation - No External Libraries
================================================================================

OVERVIEW
================================================================================
A comprehensive profanity and banned words filter implemented in pure JavaScript
without any external npm packages or libraries. The filter includes:
- Backend middleware for server-side validation
- Frontend utility for client-side warnings
- Real-time text checking
- Customizable word lists
- Censoring and validation modes

FILES
================================================================================
Backend:
  • utils/profanityFilter.js      - Core filter logic
  • middlewares/profanityFilter.js - Express middleware
  • routes/*.js                    - Integrated into routes

Frontend:
  • js/profanityFilter.js          - Client-side filter utility

BACKEND USAGE
================================================================================

1. CORE FILTER MODULE (utils/profanityFilter.js)
   
   Functions:
   
   a) checkForBannedWords(text)
      Checks if text contains banned words
      Returns: { hasBanned: boolean, violations: string[] }
      
      Example:
      const filter = require('./utils/profanityFilter');
      const result = filter.checkForBannedWords('This is damn bad');
      // Result: { hasBanned: true, violations: ['damn'] }
   
   b) censorBannedWords(text)
      Replaces banned words with asterisks (maintains word length)
      Returns: string
      
      Example:
      filter.censorBannedWords('This is damn bad');
      // Result: "This is **** bad"
   
   c) sanitizeInput(text)
      Removes and censors banned content
      Returns: string
      
   d) validateField(text, fieldName, strict)
      Validates a specific field with optional strict mode
      Parameters:
        - text: string to validate
        - fieldName: name of field (for error messages)
        - strict: boolean (false = censor, true = reject)
      Returns: { isValid, message, sanitized, violations }
      
      Example:
      filter.validateField('Hello shit', 'comment', false);
      // Result: {
      //   isValid: true,
      //   message: 'comment contained inappropriate language and has been censored.',
      //   sanitized: 'Hello ****',
      //   violations: ['shit']
      // }
   
   e) getStatistics(text)
      Get detailed stats about banned words in text
      Returns: { totalWords, bannedWordsFound, bannedWordsList, profanityPercentage }
   
   f) addBannedWords(wordsArray)
      Add custom banned words at runtime
      
   g) getBannedWordsList()
      Get all current banned words
      
   h) removeBannedWord(word)
      Remove a word from banned list

2. MIDDLEWARE (middlewares/profanityFilter.js)
   
   Functions:
   
   a) profanityFilterMiddleware(options)
      Creates middleware with custom options
      Options:
        - strict: boolean (default: false)
          * true = reject requests with profanity (400 error)
          * false = censor content and continue
        - fields: string[] (default: null - check all fields)
          * Specify which fields to check
        - allowCensored: boolean (default: true)
          * Whether to allow censored content
      
      Example:
      app.post('/comment', profanityFilterMiddleware({ strict: false }), 
               commentController.create);
   
   b) strictProfanityFilter(...fields)
      Shorthand for strict mode - rejects requests with profanity
      
      Example:
      router.post('/review', strictProfanityFilter('comment'), reviewController.add);
   
   c) censoringProfanityFilter(...fields)
      Shorthand for censoring mode - censors and allows
      
      Example:
      router.post('/review', censoringProfanityFilter('comment'), reviewController.add);

FRONTEND USAGE
================================================================================

1. INCLUDE THE SCRIPT
   
   In your HTML file:
   <script src="js/profanityFilter.js"></script>
   
   Must be after jQuery:
   <script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
   <script src="js/profanityFilter.js"></script>

2. REAL-TIME CHECKING
   
   Setup real-time profanity warnings on input fields:
   
   setupProfanityRealTimeCheck('#productName');
   setupProfanityRealTimeCheck('.comment-field');
   setupProfanityRealTimeCheck('textarea[name="description"]');
   
   This will:
   - Check text as user types
   - Show yellow warning badge if profanity detected
   - Show list of inappropriate words found

3. FORM VALIDATION
   
   Setup complete form validation:
   
   setupProfanityFormValidation('#productForm', {
       fields: 'input[type="text"], textarea',
       strictMode: false,  // true = prevent form submission
       warnings: true      // true = show real-time warnings
   });
   
   This will:
   - Show real-time warnings on all fields
   - Validate form on submission
   - Highlight fields with violations (red border)
   - Show alert if strictMode is true

4. INDIVIDUAL FUNCTIONS
   
   a) checkTextForProfanity(text)
      Returns: { hasBanned: boolean, violations: [] }
      
      Example:
      const result = checkTextForProfanity("This is damn bad");
      if (result.hasBanned) {
          alert('Inappropriate words: ' + result.violations.join(', '));
      }
   
   b) validateFormForProfanity(formSelector, strictMode)
      Returns: boolean (true = clean, false = has profanity)
      
      Example:
      if (!validateFormForProfanity('#myForm', false)) {
          console.log('Form contains profanity');
      }
   
   c) showProfanityNotification(message, type)
      Shows temporary notification popup
      Type: 'warning' or 'error'
      
      Example:
      showProfanityNotification('Your comment contains inappropriate language', 'warning');
   
   d) getProfanityStats(text)
      Returns: { totalWords, bannedWordsFound, profanityPercentage, ... }
      
      Example:
      const stats = getProfanityStats('damn this is shit');
      console.log(stats.profanityPercentage); // "25.00"

5. PRACTICAL EXAMPLES
   
   Example 1: Review form with warnings
   ====================================
   <form id="reviewForm">
       <input type="text" name="title" placeholder="Review title">
       <textarea name="comment" placeholder="Your review"></textarea>
       <button type="submit">Submit Review</button>
   </form>
   
   <script>
   setupProfanityFormValidation('#reviewForm', {
       fields: 'input, textarea',
       strictMode: false,
       warnings: true
   });
   </script>
   
   Example 2: Product description with real-time check
   ===================================================
   <textarea id="productDesc" placeholder="Product description"></textarea>
   <div id="descStats"></div>
   
   <script>
   setupProfanityRealTimeCheck('#productDesc');
   
   $('#productDesc').on('input', function() {
       const stats = getProfanityStats($(this).val());
       $('#descStats').text(
           `Words: ${stats.totalWords}, Profanity: ${stats.profanityPercentage}%`
       );
   });
   </script>
   
   Example 3: Custom form submission handler
   ==========================================
   $('#myForm').on('submit', function(e) {
       e.preventDefault();
       
       if (validateFormForProfanity('#myForm', false)) {
           // Form is clean, proceed with submission
           $.ajax({
               url: '/api/v1/reviews',
               method: 'POST',
               data: new FormData(this),
               success: function(res) {
                   alert('Review submitted!');
               }
           });
       } else {
           showProfanityNotification('Please revise your text', 'error');
       }
   });

CUSTOMIZATION
================================================================================

1. ADD CUSTOM BANNED WORDS (Backend)
   
   In your controller or middleware:
   
   const filter = require('../utils/profanityFilter');
   filter.addBannedWords(['brandname', 'competitorname']);
   
   Now 'brandname' and 'competitorname' will be filtered.

2. MODIFY BANNED WORDS LIST
   
   Backend: Edit BANNED_WORDS array in utils/profanityFilter.js
   Frontend: Edit FRONTEND_BANNED_WORDS array in js/profanityFilter.js
   
   Keep both lists in sync for consistent behavior.

3. STRICT VS. CENSORING MODE
   
   Strict Mode (reject requests):
   router.post('/admin/product', strictProfanityFilter('name', 'description'), ...);
   
   Censoring Mode (censor and allow):
   router.post('/review', censoringProfanityFilter('comment'), ...);

4. DOMAIN-SPECIFIC FILTERING
   
   For e-commerce, you might want to filter:
   - Competitor names
   - Brand names you don't allow
   - URLs (already included)
   - Phone numbers (already included in patterns)

PERFORMANCE

- Space complexity: O(m) where m = number of banned words

- Efficient regex matching for patterns

SECURITY NOTES
================================================================================

⚠️  IMPORTANT:
- Frontend filtering is for UX purposes ONLY
- Users can bypass frontend validation with developer tools
- ALWAYS validate on the backend (use middleware)
- Store sanitized content in database

Best Practice:
1. Use frontend filter for real-time user feedback
2. Use backend middleware to enforce rules
3. Combine both for best security and UX

TROUBLESHOOTING
================================================================================

Q: Middleware not working?
A: Make sure middleware is imported and placed BEFORE the route handler:
   router.post('/create', profanityFilterMiddleware(...), controller.create);

Q: Frontend warnings not showing?
A: Ensure jQuery is loaded before profanityFilter.js:
   <script src="jquery.js"></script>
   <script src="profanityFilter.js"></script>

Q: Words not being censored?
A: Check word is in BANNED_WORDS list (case-insensitive matching)
   Add custom words with: addBannedWords(['yourword'])

Q: Form still submitting with profanity?
A: Use strict mode or prevent default:
   e.preventDefault(); // in your form handler
